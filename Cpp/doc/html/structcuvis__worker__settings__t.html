◆ can_drop

CUVIS_INT cuvis_worker_settings_t::can_drop

If set to 0, the worker is not allowed to drop results from its result queue, if set to 1, dropping is allowed.

This behaviour may be desired for video playback or similar tasks where data loss is acceptable in favor of processing speed. Dropping frames should be disallowed for bulk reprocessing tasks.

◆ keep_out_of_sequence

CUVIS_INT cuvis_worker_settings_t::keep_out_of_sequence

If images are retrieved out of sequence, do not drop them (if set to 1).

Note
Typically, out-of sequence images occur, when the system can't keep up. You should reduce FPS or reduce disable processing.
